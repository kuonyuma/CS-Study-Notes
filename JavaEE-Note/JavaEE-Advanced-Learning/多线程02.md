# 前言

文本将大概介绍 **thread** 的一系列方法，如：

```java
Thread();
Thread(Runnable target);
Thread(String name);
Thread(Runnable target,String name);
isDaemon();
```

***

# 1. 多线程的引入

我们知道创建或销毁一个进程的开销是很大的。  
线程就弥补了进程的不足。

1.  在一个进程内部有多个线程，线程之间是没有关系的。
2.  一个线程出现问题可能会带走整个进程。
3.  线程共享进程的资源。

***

# 2 线程的创建

## 2.1 创建 Thread 类

> 代码如下（示例）：

```java
创建一个 Thread 类对象，
这个 API 是由官方封装好给我们的。
不需要额外的导包
Thread t = new Thread();
```

当我们 new 出来一个对象后，我们要重写一个名为 run 的方法。  
这是源码：

```java
@Override
public void run() {
    if (target != null) {
        target.run();
    }
}
```

***

## 2.2 启动线程

假设我们创建好一个线程后，如何运行呢？  
用 start  
`Thread.start()`

千万不能直接这样写 `t.run();`  
你只能对一个对象调用一次 start。  
这样只是调用一个名为 run 的方法而已，并没有创建出一个线程。

正确的方法如下：

```java
public class Main {
    public static void main(String[] args) {
        Thread t = new Thread() {
            @Override
            public void run() {
                try {
                    while (true) {
                        System.out.println(" hello Thread");
                        Thread.sleep(1000);
                    }
                } catch (InterruptedException e) {
                    throw new RuntimeException(e);
                }
            }
        };

        t.start();
        try {
            while (true) {
                System.out.println("hello main");
                Thread.sleep(1000);
            }
        } catch (InterruptedException e) {
            throw new RuntimeException(e);
        }
    }
}
```

***

## 2.3 引入 Runnable

我们期望写出一些低耦合的代码，Runnable 就可以完美实现我们的想法。  
Runnable 是一个接口，里面就只有 run 这一个抽象方法。

我们 new 一个  
`Runnable b = new Runnable();`  
重写里面的方法：

```java
@Override
public void run() {
    while (true)
        System.out.println(" hello Thread");
}
```

然后我们将这个对象传给 Thread，就有：

```java
Thread t = new Thread(b);
```

传了之后，我们再：

```java
t.start();
```

就可以在控制台输出结果了。

## 2.5 前后台线程

我们可以看到，在图中不只有两个线程，还有一些自动出现在程序中的线程。  
它们就是所谓的后台线程。

前台线程就是 main 与我们创建的“枣子姐”。

这里有几个概念需要记住：

1.  只有所有的前台线程结束运行，程序才会关闭。
2.  后台线程停止运行不会导致程序关闭。

我们可以用 `isDaemon()` 这个方法来判断这个线程是前台还是后台。

```java
Thread t = new Thread(() -> {
    System.out.println(Thread.currentThread().isDaemon());
}, "枣子姐");

t.start();
// 输出 false，表示非后台线程
```

注意：你不能直接 `t.isDaemon();`，因为 t 还没启动，你无法判断。  
你也可以这样写 `this.isDaemon();`，这个和上面那个差不多。

## 2.6 检查线程是否存活

判断一个线程是否结束，可以用 `isAlive()`。

```java
public void run() {
    for (int i = 3; i > 0; i--) {
        System.out.println(this.isAlive());
        try {
            Thread.sleep(1000);
        } catch (InterruptedException e) {
            throw new RuntimeException(e);
        }
    }
    System.out.println("线程结束");
}
```

## 2.7 中断一个线程

Thread 里面有一个字段  
`private volatile boolean interrupted;`  
它是判断线程状态的一个变量，与方法联动。

```java
public boolean isInterrupted() {
    return interrupted;
}
```

我们可以通过一个内置的方法修改这个 interrupted：

```java
public void interrupt() {
    if (this != Thread.currentThread()) {
        checkAccess();

        synchronized (blockerLock) {
            Interruptible b = blocker;
            if (b != null) {
                interrupted = true;
                interrupt0();
                b.interrupt(this);
                return;
            }
        }
    }
    interrupted = true;
    interrupt0();
}
```

有以下用法：

```java
Thread t = new Thread() {
    @Override
    public void run() {
        while (!this.isInterrupted()) {
            System.out.println(this.isAlive());
        }
        System.out.println("线程结束");
    }
};

t.start();
Thread.sleep(5000);
t.interrupt();
```

程序在 5 秒后终止。

# 总结

1.  线程相比进程开销更小，多个线程共享同一进程的资源
2.  创建线程的两种常见方式：继承 Thread 类、实现 Runnable 接口（更推荐）
3.  启动线程必须调用 start()，不能直接调用 run()
4.  可以通过构造方法给线程命名，方便调试和区分线程
5.  使用 isDaemon() 区分前台线程和后台（守护）线程
6.  使用 isAlive() 判断线程是否还在运行
7.  通过 interrupt() 和 isInterrupted() 实现线程的安全中断与结束
