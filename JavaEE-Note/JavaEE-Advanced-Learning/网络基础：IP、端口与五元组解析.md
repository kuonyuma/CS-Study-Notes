# 1. IP与端口
所谓的IP，其含义就是地址，我们经常在网络上看到的一些 ``192.168...``，这就是一个IP。
每个人的电脑都有一串属于自己的IP地址（不过这个IP并不是固定的）。
“端口”这个词语或许要更为少见一点，但其实也没那么高深。它的作用就是确定更精细的网络地址。
常见的端口号有：443、3306。端口的范围是0-65535。
举一个例子：
我在拼多多上买东西，填写的地址是：四川省乐山市市中区，乐山师范学院，XXX宿舍。
在这里，``四川省乐山市市中区，乐山师范学院``这一段就可以看作是一串IP。
而``XXX宿舍``，这就相当于一个端口号。
乐山师范学院有很多人都在买东西，为了区分包裹具体是哪一个人买的，这里就可以填上宿舍号来加以区分。
我们的电脑也是一样的道理。一台计算机上同时运行着很多应用，它们共用这台计算机的IP地址。当网络传输的数据到达电脑后，到底要往哪一个App传呢？此时，系统就是用端口号来加以区分的。
# 2. 五元组 
在 TCP/IP 协议中，**五元组**是用于唯一标识一个网络连接（Connection）的五个核心要素。
你可以把它想象成一张**“快递面单”**，没有这张面单，数据包就会在互联网中迷失方向。

---

### 五元组的构成
为了方便理解，我们可以把网络通信比作**宿舍楼之间的信件往来**：

| 要素 | 技术定义 | 生活类比（快递/寄信） | 作用 |
| :--- | :--- | :--- | :--- |
| **1. 源 IP** | 客户端的 IP 地址 | **寄件人的地址** | 标识数据包从哪台设备发出来。 |
| **2. 源端口** | 客户端进程的端口号 | **寄件人的具体姓名** | 同一个宿舍（IP）住着很多人，端口号决定是谁寄的。 |
| **3. 目的 IP** | 服务器的 IP 地址 | **收件人的地址** | 标识数据包要发到哪台服务器。 |
| **4. 目的端口** | 服务器进程的端口号 | **收件人的具体姓名** | 同一个宿舍（IP）有很多人，端口号决定给谁。 |
| **5. 协议** | 通信协议（TCP/UDP） | **快递公司 / 运输方式** | 双方约定的规则（如顺丰或邮政），必须统一标准。 |

#### 1. 源 IP (Source IP)
这是发起连接方的互联网身份标识。
你的手机或电脑当前的 IP 地址。

#### 2. 源端口 (Source Port)
表示数据由客户端的哪一个进程（程序）发送。
通常由操作系统随机分配。就像宿舍里有张三、李四.
源端口决定了这封信是张三写的。

#### 3. 目的 IP (Destination IP)
这是接收方（通常是服务器）的身份标识。
你要访问的百度或腾讯服务器的公网 IP。

#### 4. 目的端口 (Destination Port)
表示数据由服务器上的哪一个服务进程接收。
Web 服务通常是 `80` 或 `443` 端口，邮件服务通常是 `25` 端口。

#### 5. 协议 (Protocol)
标识双方通信时遵循的规则。
常见类型：**TCP**或 **UDP**。

# 3. 协议
我的朋友小帅，人如其名，长得确实不赖。
某天，小帅在交友 App 上认识了一位漂亮的小姐姐，网名“小美”。
随着时间的推移，两人聊得愈发投缘，从最初的 App 私聊到加上微信，最后甚至打起了视频通话。
那场面，用现在的话说，聊得可谓是“十分甚至九分”的火热。

终于，小帅按捺不住寂寞，给小美发了一条消息：

“我们要不约个地方见面吧？就在东大桥底下的天桥下面。”

小美欣然同意。

到了约定的日子，小帅把自己打扮得十分精神，西装领带红玫瑰。
他准时赶到天桥附近，果然看见一位美女正站在桥下。
小帅满心欢喜地走过去，两人接上头后便边走边聊。
小帅发现，小美线下的真实样子比照片好看多了，倾国倾城都不会过。
双发也是交谈甚欢。

但聊着聊着，小帅越发感觉不对劲。他忍不住问了一句：

“那个……你的网名是叫‘东北御姐’吗？”

美女愣住了：“不是啊，我是来等人的。你的网名难道叫‘EXO 最后的王’？”
小帅当场懵圈——认错人了。

情况是这样的：
这位姑娘也是和另一位男生约好在天桥见面，结果时间、地点全撞车了。
小帅思索片刻，尴尬地留下一句“抱歉（じゃあな）”，赶紧折返回去找小美。

那天风很大，也很冷。姑娘的长发在风中飘荡，默默地望着小帅奔跑远去的背影。
当小帅气喘吁吁地跑回原位时，天桥下方早已空无一人……
回去后，小帅再也没收到过小美的微信，而那位桥下的姑娘，也早已消失在了人潮之中。

从上面的故事可以看出，要是小帅和小美双方规定一个暗号就不会发生这样的故事了，
比如这天小帅穿马甲，小美穿白色裙子之类的。

这样的暗号就叫做协议。
这里我们给出规整的定义：

协议，是网络协议的简称。
网络协议是网络通信（即网络数据传输）过程中，所有经过的网络设备都必须共同遵守的一组约定和规则，例如如何建立连接、如何相互识别等。
只有遵守这些约定，计算机之间才能相互通信交流。协议（protocol）最终体现为在网络上传输的数据包格式。

计算机之间的传输媒介是光信号和电信号，通过“频率”和“强弱”来表示 0 和 1 这样的信息。要想传递各种不同的信息，就需要约定好双方的数据格式。

- 计算机生产厂商有很多；
- 计算机操作系统也有很多；
- 计算机网络硬件设备也有很多；

如何让这些不同厂商生产的计算机能够顺畅通信？  
就需要有人站出来，约定一个共同的标准，大家都来遵守，这就是网络协议。

# 4. 协议分层

网络世界错综复杂，不同的交流之间有不同的协议，
为了通信方便，我们通常将协议给封装起来，以API的形式提供给对方，让对方使用。

![在这里插入图片描述](https://i-blog.csdnimg.cn/direct/f54331695699404ca630f82afef4a1e0.png)



我们可以把打电话分成**上下两层**来看：

*   **上面这层叫“语言层”：** 这是**人与人**之间的事。A君和B君聊天，只要大家都说汉语（这就是**协议**），就能聊得通。
*   **下面这层叫“设备层”：** 这是**机器与机器**之间的事。两部电话之间通过电线、信号相连，把声音传过去。
*   **中间的绿箭头叫“接口”：** 就是人对着话筒说话，耳朵听听筒，它是人和机器连接的地方。

1.  **换手机不用学外语（下变，上不变）：**
    就算你把手里的老古董电话换成了最新的 iPhone（设备换了），你依然可以用汉语聊天。你不需要为了换个手机，专门去学一门新语言。
2.  **说外语不用换手机（上变，下不变）：**
    如果你和朋友决定改用英语聊天（语言换了），手里的电话还是原来的那个，不需要因为换了语种就跑去商场买部新电话。

# 总结 

1. IP地址定位了网络中的主机位置（到哪栋楼），端口号精确定位了具体的应用程序（到哪个宿舍的哪个人）。
2. “源IP、源端口、目的IP、目的端口、协议”共同构成的五元组，就像是一张完整的快递面单，在复杂的网络中唯一标识了一条通信连接。
3. 在五花八门的计算机硬件和操作系统之间，协议就是大家事先约定好的规则与暗号。只有遵守同一个标准格式，不同设备之间才能听懂彼此的“光电信号”，实现顺畅交流。
4. 通过将复杂的网络通信拆分为不同的独立层级，并定义好层与层之间的接口。这种设计让上下层互不影响，极大地提高了扩展性——也就是所谓的“换新手机不用重新学外语，换新语言沟通也不用买新手机”。
